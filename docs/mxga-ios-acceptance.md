# MXGA Userscript iOS / iPadOS 真机验收

这份清单用于验证 `scripts/make-x-great-again.user.js` 在 App Store 版 Userscripts + Safari 真机环境中的兼容性。模拟器、桌面 WebKit、Node 测试和“能安装”都不能替代本清单。

## 准备

- 安装链接：<https://raw.githubusercontent.com/kyangc/tampermonkey_scripts/main/scripts/make-x-great-again.user.js>
- 关闭原版 MXGA Safari / 浏览器扩展，避免重复徽标和两套隐藏记录。
- 在“设置 → Safari → 扩展 → Userscripts”中启用扩展，并允许访问 `x.com`。
- 在 Userscripts 中使用 `New Remote` 添加上面的 raw URL，确认脚本已启用。
- 测试公开账号时，从 <https://x.zuoluo.tv/list> 选择一个公榜账号；不要用私人时间线截图作为证据。

Userscripts 4.x 的可选编辑器 linter 可能把 `@namespace`、`@license`、`@homepageURL`、`@supportURL`、`@connect` 和 `@source` 标为不支持的扩展 metadata；这些字段不参与 Userscripts 运行时注入，正常情况下不会阻止保存。五个 `GM.*` grant 必须全部被识别为受支持；如果 Save 被禁用或脚本无法启用，仍视为失败。

## 必须通过

### 1. 安装与更新

- [ ] `New Remote` 能保存脚本，没有 metadata 或 grant 阻断错误。
- [ ] Userscripts 能识别 `@version`、`@updateURL` 和 `@downloadURL`。
- [ ] Safari 打开 `https://x.com/` 后，页面右下区域出现 `MXGA` 控制按钮。

记录：安装是否需要额外操作、Userscripts 是否只显示不影响保存的 metadata linter 提示。

### 2. 首次名单同步与内存稳定性

- [ ] 首次进入 X 后能完成名单同步，控制面板显示名单数量、版本和同步时间。
- [ ] 同步过程中 Safari 标签页没有白屏、自动重载、崩溃或被系统回收。
- [ ] 同步完成后连续滚动首页或评论区 3 分钟，没有明显持续卡顿。
- [ ] 退出到主屏幕 5 分钟再回到 Safari，页面仍能继续工作；若系统重载页面，记录重载发生时间。

记录：从打开页面到名单就绪的秒数、面板显示的名单数量和版本、是否发生标签页重载。

### 3. 名单徽标与安全默认值

- [ ] 打开一个公榜账号的 X 个人主页，能看到 MXGA 徽标。
- [ ] 点击徽标后能看到类别和“人工确认”或“自动收录”层级。
- [ ] 未点击“本地隐藏”前，脚本不会自动隐藏该账号的帖子。
- [ ] 控制面板明确说明不会执行 X 原生静音或拉黑。

### 4. 触控与响应式布局

- [ ] 轻点徽标能打开详情弹层，不依赖鼠标悬停。
- [ ] 弹层完整位于屏幕内，“本地隐藏 / 误判申诉 / 关闭”均可点击。
- [ ] 竖屏和横屏各打开一次详情与设置面板，没有按钮被 Safari 工具栏或 X 底部导航遮挡。
- [ ] iPad 分屏模式下，窄窗口仍可滚动设置面板并操作恢复按钮。

### 5. 隐藏、撤销与持久化

- [ ] 点击“本地隐藏”后，同一账号当前可见的帖子消失。
- [ ] 5 秒内点击“撤销”，帖子恢复。
- [ ] 再次隐藏并刷新页面，该账号帖子仍保持隐藏。
- [ ] 从控制面板的“本地隐藏记录”点击“恢复”，帖子重新显示且记录被删除。
- [ ] 打开第二个 X 标签页，重新聚焦后能读取同一份隐藏记录。

### 6. 同步失败降级

- [ ] 名单已成功同步一次后，临时切换飞行模式并刷新 X，脚本继续使用本地缓存。
- [ ] 在飞行模式下点击“立即更新名单”，面板显示更新失败，而不是清空已有名单或白名单。
- [ ] 恢复网络后点击“立即更新名单”可以恢复正常状态。

## 通过标准

只有以上所有“必须通过”项目在至少一台真实 iPhone 或 iPad 上通过，才可以把 iOS Userscripts 兼容性标记为完成。首次大名单同步发生白屏、标签页重载或崩溃，视为内存门槛未通过，不能以第二次缓存加载成功替代。
